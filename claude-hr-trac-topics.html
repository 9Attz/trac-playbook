Topic 01

CRAFT Deep Dive

เจาะลึกแต่ละองค์ประกอบของ CRAFT พร้อมตัวอย่างจริงสำหรับงาน HR ของ TRAC

C

Context — บริบทที่ครบถ้วน

บอก Claude ทุกอย่างที่จำเป็น: บริษัทอะไร, ทำธุรกิจอะไร, พนักงานกี่คน, ผู้อ่านคือใคร, วัตถุประสงค์คืออะไร — ยิ่งครบ ยิ่งได้คำตอบที่ตรง

❌ "เขียน policy ลาพักร้อน" ✅ "บริษัท TRAC ให้เช่ารถ พนักงาน 500 คน มีทั้งพนักงานออฟฟิศและพนักงานขับรถ ช่วยเขียน Annual Leave Policy ที่เหมาะกับทั้ง 2 กลุ่ม สอดคล้องกับกฎหมายแรงงานไทย"
R

Role — บทบาทที่ชัดเจน

กำหนดบทบาทให้ Claude ได้รับทราบว่าควรตอบในฐานะใคร เพิ่มความเชี่ยวชาญและมุมมองที่ถูกต้อง

❌ ไม่กำหนด role ✅ "คุณเป็นที่ปรึกษากฎหมายแรงงานไทยที่มีประสบการณ์ 20 ปี เชี่ยวชาญในอุตสาหกรรม Logistics และ Fleet Management"
A

Action — สิ่งที่ต้องการชัดเจน

ระบุ action ที่ต้องการให้ชัด อย่าให้ Claude ต้องเดาว่าต้องการอะไร ใช้ verb ที่ชัดเจน: เขียน, วิเคราะห์, สรุป, เปรียบเทียบ, ร่าง

❌ "ช่วยด้วยเรื่อง JD" ✅ "เขียน Job Description ตำแหน่ง Fleet Operations Manager โดย: 1) เขียน responsibilities 8-10 ข้อ 2) ระบุ qualifications แยก required/preferred 3) เพิ่ม DEI language"
F

Format — รูปแบบที่ต้องการ

บอก output format ที่ต้องการ: ตาราง, bullet points, numbered list, เขียนเป็น paragraph, email format, หรือ ความยาวที่ต้องการ

❌ "สรุปให้หน่อย" ✅ "สรุปเป็นตาราง 3 คอลัมน์: หัวข้อ / ประเด็นสำคัญ / Action ที่ต้องทำ ความยาวไม่เกิน 1 หน้า A4"
T

Tone — โทนที่เหมาะสม

กำหนด tone ของการตอบ: ทางการ, เป็นกันเอง, ภาษาธุรกิจ, ภาษาสำหรับพนักงาน, ภาษาสำหรับผู้บริหาร

❌ ไม่ระบุ tone ✅ "ภาษาไทยทางการ เหมาะสำหรับประกาศบริษัท — ไม่ formal เกินไป อ่านแล้วเป็นกันเองแต่ยังดูมืออาชีพ"

Topic 02

Prompt Patterns ที่ใช้บ่อย

รูปแบบ prompt ที่ HR มืออาชีพใช้บ่อยที่สุด copy ไปปรับใช้ได้เลย

Pattern: Before/After

ปรับปรุงของเดิม

แนบเอกสารเดิม ขอให้ Claude ปรับปรุงตามเกณฑ์ที่กำหนด

นี่คือ JD เดิมของเรา [แนบไฟล์] ช่วยปรับปรุงโดย: 1. เพิ่ม DEI language 2. ลด jargon ที่ไม่จำเป็น 3. เพิ่ม Employer Branding ของ TRAC (คง structure เดิมไว้)

Pattern: Multiple Options

ขอหลายตัวเลือก

ให้ Claude สร้างหลาย version แล้วเลือกที่ดีที่สุด

เขียน email ประกาศนโยบาย WFH ใหม่ ให้ 3 version: - Version A: ทางการ (สำหรับ Board) - Version B: กลาง (สำหรับ Manager) - Version C: เป็นกันเอง (สำหรับพนักงาน) แต่ละ version ไม่เกิน 150 คำ

Pattern: Devil's Advocate

ให้ Claude วิจารณ์งานตัวเอง

หลังได้ผลลัพธ์แล้ว ให้ Claude ชี้จุดอ่อนและปรับปรุง

นี่คือ policy ที่เราร่างไว้ [วาง text] ช่วยชี้จุดอ่อนที่: 1. อาจขัดกฎหมายแรงงาน 2. พนักงานอาจเข้าใจผิด 3. ฝ่าย Legal อาจมีปัญหา แล้วแนะนำการแก้ไข

Pattern: Step-by-Step Guide

ขอ process ทีละขั้น

ให้ Claude อธิบาย process ที่ซับซ้อนแบบ step-by-step ที่ทุกคนทำตามได้

อธิบาย process การทำ Performance Review แบบ step-by-step สำหรับ Manager ใหม่ ที่ไม่เคยทำมาก่อน ใส่: timeline, เอกสารที่ต้องใช้, คำถามตัวอย่าง, ข้อควรระวัง ภาษาไทย เข้าใจง่าย

Topic 03

Common Mistakes ที่ต้องเลี่ยง

ความผิดพลาดที่พบบ่อยที่สุดเมื่อ HR เริ่มใช้ Claude — และวิธีแก้ไข

❌ ผิด

"เขียน policy ให้หน่อย"

คำสั่งสั้นเกินไป ไม่มี context Claude ไม่รู้ว่าต้องการอะไรกันแน่

✅ ถูก

"คุณเป็นที่ปรึกษา HR เขียน Annual Leave Policy สำหรับ TRAC (บริษัทเช่ารถ 500 คน) สอดคล้องกฎหมายแรงงานไทย รูปแบบ: หัวข้อ, นิยาม, หลักเกณฑ์, ข้อยกเว้น"

❌ ผิด

ใส่ชื่อจริง-เงินเดือนจริงของพนักงาน

ละเมิด PDPA และอาจเป็นความเสี่ยงด้านความปลอดภัยของข้อมูล

✅ ถูก

ใช้ข้อมูล anonymized เช่น "พนักงาน A, B, C" หรือ "เงินเดือนระดับ 1, 2, 3" หรือ "ประมาณ 25,000-30,000 บาท" แทนตัวเลขจริง

❌ ผิด

ใช้ผลลัพธ์จาก Claude โดยไม่ตรวจสอบ

Claude อาจผิดพลาดด้านกฎหมาย ตัวเลข หรือข้อมูลเฉพาะบริษัท

✅ ถูก

ใช้ Claude เป็น "first draft" เสมอ แล้ว review ด้วยตัวเองหรือผู้เชี่ยวชาญก่อนใช้งานจริง โดยเฉพาะเรื่องกฎหมายแรงงาน

❌ ผิด

เลิกใช้เมื่อได้คำตอบที่ไม่ดีครั้งแรก

คำตอบแรกไม่ได้ถูกต้องเสมอไป — Claude ต้องการ feedback เพื่อปรับปรุง

✅ ถูก

ให้ feedback ต่อ: "เพิ่มรายละเอียดด้านนี้", "ปรับโทนให้เป็นทางการขึ้น", "เขียนใหม่โดยเน้น..." — iterate จนได้ผลที่ต้องการ

Topic 04

Advanced Techniques

เทคนิคขั้นสูงที่จะยกระดับการใช้ Claude จาก intermediate สู่ Pro

Technique 01

Chain Prompting — ต่อกันหลาย prompt

แทนที่จะถามครั้งเดียวยาวๆ ให้แบ่งเป็นขั้น: Step 1 ขอ outline → Step 2 ขอ draft ทีละ section → Step 3 ขอ review และปรับปรุง — ได้ผลลัพธ์ดีกว่ามาก

Technique 02

Persona Switching — เปลี่ยน role ตามโจทย์

ใช้ role ที่ต่างกันสำหรับงานต่างกัน: "ที่ปรึกษากฎหมาย" สำหรับ policy review, "นักจิตวิทยาองค์กร" สำหรับ engagement, "นักสื่อสารมืออาชีพ" สำหรับ internal comms

Technique 03

Reference Upload — ให้ดูตัวอย่างจริง

อัพโหลดตัวอย่างที่ชอบ แล้วบอก Claude ว่า "เขียนแบบนี้แต่เนื้อหาเป็น..." — Claude จะจับ style, tone และ format ได้แม่นยำกว่าการอธิบายเอง

Technique 04

Constraint Setting — กำหนดข้อจำกัด

บอกสิ่งที่ ไม่ต้องการ ด้วย: "อย่าใช้ภาษาที่ exclusive", "ห้ามยาวเกิน 1 หน้า", "ห้ามใช้ jargon ที่คนทั่วไปไม่รู้จัก" — ช่วยควบคุม output ได้ดีขึ้น

Technique 05

Ask Claude to Ask You — ให้ Claude ถามก่อน

เพิ่มประโยค "ถ้าต้องการข้อมูลเพิ่มเติมเพื่อตอบได้ดีขึ้น ถามฉันก่อนได้เลย" — Claude จะถามคำถามที่ช่วยให้ได้ผลลัพธ์ที่ตรงความต้องการมากขึ้น
