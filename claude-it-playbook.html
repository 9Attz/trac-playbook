02 — Fundamentals

พื้นฐานที่ ต้องเข้าใจ

ก่อนใช้ Claude ทำงาน IT ได้ดี ต้องเข้าใจหลักการ 4 ข้อนี้ก่อน — คนส่วนใหญ่ข้ามขั้นนี้ไป แล้วได้ผลลัพธ์ไม่ตรงต้องการ

Context is Everything

บอก Claude ให้ครบก่อนเสมอ: OS อะไร, version อะไร, error message ทั้งหมด, สิ่งที่ลองทำไปแล้ว — ยิ่งให้ข้อมูลมาก ยิ่งได้คำตอบที่แม่นยำกว่า

Debug = Iteration

ไม่ต้องได้คำตอบสมบูรณ์ครั้งแรก ให้คุยต่อเนื่อง แจ้ง error ที่เกิดจาก solution ก่อนหน้า แล้วให้ Claude ช่วย debug ต่อไปทีละขั้น

Role = Expert Quality

กำหนดบทบาทให้ Claude เช่น "คุณเป็น Senior SysAdmin ที่เชี่ยวชาญ Windows Server 15 ปี" จะได้คำตอบที่ลึก เฉพาะทาง และเป็นประโยชน์กว่ามาก

Output Format Matters

บอกรูปแบบที่ต้องการชัดเจน — step-by-step, script พร้อม comment, runbook, checklist, หรือ table เพื่อให้ได้ output ที่ใช้งานได้ทันที

03 — Prompting

เทคนิค Prompting สำหรับ IT

สูตรการพิมพ์คำสั่งที่ได้ผลลัพธ์ดี ใช้ได้กับทุกงาน IT Admin และ IT Support

Formula

สูตร CRAFT Prompt

C — Context: บอกบริบท (OS อะไร, version, environment, error message ทั้งหมด)
R — Role: กำหนดบทบาท (Senior SysAdmin, Security Engineer, Network Admin)
A — Action: บอกสิ่งที่ต้องการ (troubleshoot, เขียน script, สร้าง documentation)
F — Format: ระบุรูปแบบ output (step-by-step, PowerShell script, runbook, checklist)
T — Tone: ภาษาที่ใช้ (ภาษาไทย, English, technical, ภาษาสำหรับ end-user)

ตัวอย่าง: Troubleshoot DNS Issue

Prompt ที่ดี

คุณเป็น Senior Network Admin ที่มีประสบการณ์ 10 ปี สถานการณ์: Windows Server 2022, DNS Role ติดตั้งอยู่ ปัญหา: Client บางเครื่องใน subnet 192.168.10.0/24 resolve hostname ไม่ได้ Error: "DNS request timed out" เมื่อ ping ด้วย hostname สิ่งที่ลองแล้ว: restart DNS service, flush DNS cache บน client ช่วย: 1. วิเคราะห์สาเหตุที่เป็นไปได้ทั้งหมด 2. ให้ checklist troubleshoot แบบ step-by-step 3. ระบุ command ที่ต้องรันและผลที่คาดหวัง
Pro Tip: อัพโหลด log file (Event Viewer, DNS debug log) มาพร้อมกัน Claude จะวิเคราะห์ได้แม่นยำกว่าการพิมพ์อธิบายเอง ประหยัดเวลาได้มาก

ตัวอย่าง: เขียน Script

Prompt สร้าง PowerShell Script

คุณเป็น PowerShell expert เขียน script สำหรับ Windows Server 2019 สร้าง script ที่: - ตรวจสอบ disk space ทุก drive (threshold: ต่ำกว่า 20%) - ส่ง email alert ผ่าน SMTP เมื่อ disk ต่ำกว่า threshold - Log ผลลัพธ์ใส่ไฟล์ .txt พร้อม timestamp - รองรับการกำหนด [server list] จาก parameter ใส่ comment อธิบายทุก section และ error handling ด้วย

05 — How-To

วิธีเริ่มต้น Step by Step

ทำตาม 6 ขั้นตอนนี้ ใช้ Claude งาน IT ได้ทันทีภายใน 10 นาที

A. เริ่มต้นใช้ Claude.ai สำหรับ IT

1

สมัคร Account

ไปที่ claude.ai สมัครด้วย Google หรือ email ใช้ฟรีได้เลย (มีโควต้าจำกัด) หรือ Pro $20/เดือน สำหรับงาน IT หนักๆ แนะนำ Pro

2

สร้าง IT Projects

กด Projects → สร้าง project แยกตามระบบ เช่น "Windows Server", "Network", "Helpdesk" แล้วใส่ Custom Instructions เช่น "ฉันเป็น IT Admin ดูแล Windows Server 2019, VMware, Cisco switch"

3

อัพโหลด Knowledge Base

ใน Project ให้อัพโหลดไฟล์พื้นฐาน: Network diagram, IP scheme, Server inventory, Policy ปัจจุบัน — Claude จะใช้เป็น context ให้คำตอบที่ตรงกับ environment จริงของคุณ

4

แนบ Log / Error ทุกครั้ง

เมื่อ troubleshoot ให้แนบ log file, screenshot error, หรือ config file มาพร้อมกัน Claude อ่าน PDF, text, ภาพ ได้หมด — ดีกว่าพิมพ์ copy-paste มาก

5

ใช้สูตร CRAFT ถามครั้งแรก

ระบุ Context (OS, version, error), Role (Senior SysAdmin), Action (troubleshoot/เขียน script), Format (step-by-step/script), Tone (ภาษาไทย) แล้วดูผลลัพธ์

6

Save Prompt ที่ได้ผลดี

Prompt ที่ใช้งานได้จริง copy เก็บไว้ใน Notepad หรือ Notion ใช้ซ้ำได้เลย สร้าง prompt library ของทีมประหยัดเวลาได้มาก

B. ใช้ Cowork สร้าง IT Dashboard

1

เปิด Cowork

ไปที่ claude.ai/cowork เปิด workspace ใหม่ — ไม่ต้อง install อะไร ทำงานได้ทันทีบนเบราว์เซอร์

2

อัพโหลดข้อมูล IT

Drag & drop ไฟล์ Excel/CSV เช่น Server inventory, Ticket log, Incident report — Claude อ่านได้ทันที

3

สั่ง Claude สร้าง Dashboard

พิมพ์: "สร้าง IT Operations Dashboard จากข้อมูลนี้ แสดง ticket volume, MTTR, system uptime, top issues by category" แล้วรอดูผล

4

Publish & Share

กด Publish ได้ link แชร์ให้ทีม หรือผู้บริหาร ดูได้ทันที ไม่ต้อง install PowerBI หรือ Grafana

06 — Tips & Best Practices

เคล็ดลับ ใช้ให้เก่ง

เทคนิคที่จะทำให้คุณใช้ Claude สำหรับงาน IT ได้เหนือกว่าคนทั่วไป

🔒

ห้าม Share Credential จริง

ห้ามใส่ password, API key, private IP ของจริง ให้ใช้ placeholder เช่น [PASSWORD], [API_KEY] แทนเสมอ

📎

แนบ Log ทุกครั้ง

Claude อ่าน log file, config file, screenshot error ได้หมด แนบมาดีกว่าพิมพ์อธิบาย — ได้คำตอบแม่นกว่ามาก

🗂️

ใช้ Projects แยกตามระบบ

สร้าง Project แยก: "Windows Server", "Network", "Helpdesk", "Security" — Claude จำ context ให้ ไม่ต้องอธิบายซ้ำทุกครั้ง

✅

Test Script ก่อน Run จริงเสมอ

Script ที่ Claude เขียนให้ ต้อง review และ test บน lab/dev environment ก่อนเสมอ อย่า run บน production โดยตรง

🇹🇭

พิมพ์ไทยได้เลย

Claude เข้าใจภาษาไทยดีมาก พิมพ์ไทยล้วน ไทยปนอังกฤษ หรืออังกฤษล้วนก็ได้ ตอบ comment code เป็นไทยได้ด้วย

🧠

ให้ Claude อธิบาย Reasoning

เพิ่ม "อธิบายเหตุผลด้วยว่าทำไมถึงเลือกวิธีนี้" จะได้ solution ที่เข้าใจ ปรับแต่งต่อได้ ไม่ใช่แค่ copy-paste

กฎเหล็ก Security

Anonymize ข้อมูลก่อนทุกครั้ง

ข้อมูล sensitive ของระบบ (IP จริง, hostname จริง, credential, customer data) ให้ แทนที่ด้วย placeholder ก่อนส่งให้ Claude เสมอ เพื่อป้องกันการ leak ข้อมูล โดยเฉพาะถ้าบริษัทมี data compliance policy

Prompt Templates สำหรับ IT (Copy ไปใช้ได้เลย)

Template: Analyze Error Log

คุณเป็น Senior SysAdmin ที่เชี่ยวชาญ [Windows Server/Linux] วิเคราะห์ log ที่แนบมา โดย: 1. ระบุ error หลักและ root cause ที่น่าจะเป็น 2. เรียง priority ของปัญหาตามความรุนแรง 3. ให้ step-by-step การแก้ไขแต่ละปัญหา 4. ระบุ preventive measure เพื่อไม่ให้เกิดซ้ำ (ตอบเป็นภาษาไทย)

Template: Write PowerShell Script

คุณเป็น PowerShell expert เขียน script สำหรับ Windows Server [version] Task: [อธิบายสิ่งที่ต้องการ] Requirements: - รองรับ parameter input - มี error handling และ try/catch - Log ผลลัพธ์พร้อม timestamp - Comment อธิบายทุก section เป็นภาษาไทย - ทดสอบได้บน [environment]

Template: Write KB Article

คุณเป็น IT Support ที่เขียน Knowledge Base สำหรับ end-user ทั่วไป เขียน KB article เรื่อง: [หัวข้อปัญหา] รูปแบบ: - ชื่อบทความ (เข้าใจง่าย ไม่เป็น technical) - อาการที่พบ (bullet points) - สาเหตุที่พบบ่อย - วิธีแก้ไขด้วยตัวเอง (step-by-step พร้อมรูป placeholder) - เมื่อไรควรติดต่อ IT (ภาษาไทย ระดับ non-technical user)
